0. 한 줄 결론

퀄리티 격차는 모델이 아니라 규율에서 온다. 잘 뽑는 사람들은 ①스타트프레임에 공을 다 쏟고 ②샷당 여러 테이크를 뽑아 골라내고 ③조명·카메라를 촬영감독 언어로 지시하고 ④후반에서 그레인·그레이드로 "AI 광택"을 벗겨내고 ⑤사운드를 3레이어로 쌓는다. 우리는 이 5개를 전부 생략하거나 절반만 하고 있었다.

1. 격차 진단 — 우리 vs 잘 뽑는 사람들

# 항목 우리 현재 프로 관행 근거
1 테이크 수 1테이크, 실패시만 재롤 샷당 ~3테이크 생성 → 체리픽. keep rate ~25% (164클립 생성/41 사용). 단편 1편에 3,229 생성/242시간 △ Invideo 가이드, Creative Bloq "Catacombs" 사례
2 스타트프레임 soul 0.12cr 스틸 그대로 I2V "스타트프레임이 포토리얼 결과의 단일 최대 결정 요인. 약하면 캐릭터가 인공적"(Catacombs 감독 원문 ✅). Kling 공식 스택도 스틸→모션→업스케일러 순 ✅ ✅ 사례 원문+Kling 공식
3 조명 언어 시간대 락 정도 (석양/한낮) 조명 서술이 단일 요소 중 품질 임팩트 최대 (Seedance 공식). 조명원+질감 명시: "single side-light", "golden hour", "practical only" ◐ BytePlus Seedance 2.0 공식 문서

6 사운드 음악 1트랙 or 무음 (젠이츠 제외) 앰비언스 베드 → 액션별 SFX → 음악 3레이어 + 내레이션 덕킹 △ SFX Engine (업계 표준 관행)
7 네이티브 오디오 sound:off로 버림 Veo 3.1 오디오 프롬프트 문법 / Kling 3.0 Omni 한·일 대사+립싱크 ✅ Veo / ◐ Kling

우리가 이미 프로 관행과 일치하는 것 (유지): I2V 모션 전용 프롬프트(이미지 재서술 금지), 정체성/모션 분리(Soul ID), 프레임 단위 검수 게이트, 오염 클립 트림 살리기(salvage), 원본 재고정(체이닝 금지), 컷리듬/인비저블 컷/3초 훅.

2. 모델별 프롬프트 공식 (공식 문서 기준)

Veo 3.1 ✅ (미사용 — 도입 후보)

  • 공식 5부 공식: [Cinematography] + [Subject] + [Action] + [Context] + [Style & Ambiance] — 카메라 먼저.
  • 40~100단어 권장, 150+ 넘으면 초점 상실.
  • 필름 에뮬레이션 토큰 실효: "shot as if on 1980s color film, slightly grainy".
  • 오디오 프롬프트 문법: 대사=따옴표(A woman says, "..."), 효과음=SFX:, 배경음=Ambient noise: — 프롬프트 안에서 사운드 레이어 연출.
  • 타임스탬프 프롬프팅: [00:00-00:02] ... [00:02-00:04] ...로 한 번 생성에 멀티샷 시퀀스 연출.
  • 스펙: 720p/1080p(+4K), 9:16 지원, 4/6/8초. 단 1080p/4K는 8초 필수.
  • 네거티브 없음 — 빼고 싶으면 원하는 장면을 구체 서술 ("no man-made structures" 대신 "a desolate landscape with no buildings or roads").
  • 시작+끝 프레임 키프레이밍(오디오 포함), 공식 워크플로 = 이미지 모델로 양 키프레임 완성 후 애니메이트.

Kling 3.0 ✅ (우리 주력)

  • 공식 요소 순서: subject, action, setting, camera language, lighting, mood — 평문 장면 연출.
  • 자연어 카메라 어휘 실효: close-up, wide shot, low angle, slow push-in, pan, tilt, tracking shot.
  • "보이는 모션 언어" 원칙: 시청자 눈에 보이는 것 서술 > 내부 기술 용어.
  • 공식 예시도 품질 토큰 스택 사용: "Photorealistic, 8K detail, masterpiece cinematography" + "god rays" + "atmospheric haze".
  • 스틸 프롬프트 슬롯 ✅: [Character]+[Pose/Action]+[Environment]+[Camera/Lens]+[Lighting/Color]+[Style/Quality]+[Timing] + 모션 큐 슬롯 "35mm lens, shallow depth of field, slow dolly in, 6-second move" (스틸→I2V로 이월). 표현 흔들림 금지("brown"이냐 "brunette"냐 하나만) — 템플릿으로 저장해 샷마다 복제.
  • 시작+끝 프레임 ✅: 두 이미지 최대한 유사해야 함, 전환 5초 이내. 차이 크면 "렌즈 스위치"(의도 없는 컷) 발생.
  • 시드 규율 ✅: 시드 고정 → 변수 1개씩만 변경 → 파일명+CSV(shot/seed/cfg/lens/lighting) 기록 → "골든 샘플" 폴더.
  • 네버리스트 ✅: extra fingers/warped hands/asymmetry/color shift/watermark/over-sharpening 등 상시 네거티브를 버전 관리하며 전 렌더에 재사용.
  • Kling 3.0 Omni 네이티브 오디오 ✅: 대사+SFX+앰비언스 단일 생성, 한국어·일본어 포함 5개 언어, 립싱크·사투리·한 씬 다국어 대사 — 캐릭터 대사에 TTS 오버레이 대안.
  • Kling 네이티브 4K ✅: kling.ai 자체 4K 모드 30크레딧/초(그쪽 크레딧 기준). 힉스필드에선 kling3_0 mode:'4k'로 노출 — 비용 확인 후 히어로 편에만.
  • Kling 공식 권장 스택 ✅: T2I 1개(스타일) + I2V 엔진 1개(모션) + 업스케일러/보간기 1개(해상도·케이던스) + 오디오 툴 1개 — 우리 후반 패스 방향과 일치.

Seedance 2.0 ✅ (힉스필드 현 추천 1순위 — 미사용. BytePlus 공식 문서 인용 + 복수 가이드 교차 확인)

  • 공식 8요소 순서: 주체 + 동작 디테일 + 씬/환경 + 조명·색톤 + 카메라 무빙 + 비주얼 스타일 + 화질 + 제약어, 50~150단어.
  • 복잡한 영상은 Shot 1 / Shot 2 / Shot 3 스토리보드 라벨로 쓰고 병합 (한 덩어리 카피라이팅 금지).
  • 카메라는 한 샷에 1무빙만 — push+pan+move 조합은 불안정 유발. 초 단위 타이밍 지정("0–3초에...")도 불안정.
  • 동작은 신체 부위 수준 + 범위/속도/힘 정량화. 고폭발 동작(전력질주·큰 점프·구르기)보다 느리고 부드러운 연속 동작 우선.
  • 감정은 추상어("very sad") 금지 → 물리 디테일로 외재화.
  • 얼굴 드리프트 공식 해법: 얼굴만 있는 클로즈업 헤드샷 ref 추가(배경 최소) + 프롬프트에 주체-이미지 바인딩, 제일 중요한 에셋을 첫 번째로. 3면도/멀티뷰 캐릭터 시트는 금지 — 딴 사람 여럿으로 인식해 쌍둥이 아티팩트 악화.
  • 유령 자막: 프롬프트로 100% 차단 불가. 세로가 가로보다 자막 확률 높음 → 가로로 생성 후 세로 크롭이 공식 우회.
  • 비용 규율(△ Catacombs): fast 저화질로 프롬프트 확정 → std 고품질로 다중 테이크 → 체리픽.
  • 힉스필드 자체 Seedance 가이드 ✅(2026-07-21 확인): 샷 수·총 길이·비율을 프롬프트 최상단에 선언. 연속 원테이크는 "No cuts, no zoom, natural head movement" 명시(안 쓰면 멋대로 컷 전환). 리얼리즘 강제 = "no 3D, no cartoon, no VFX". VFX는 인라인 브래킷 [VFX: ...], 슬로모 마커 "RAMPS TO SLOW MOTION"/"SNAPS BACK".
  • 힉스필드 스펙: std 모드에서 1080p/4K, bitrate_mode:high, genre 힌트(action/drama/epic...), image/video/audio 레퍼런스 입력.

Higgsfield Cinema Studio ✅ (우리 플랫폼 자체 기능 — 아래 4개 공식 블로그 원문 확인)

  • 카메라 어휘가 실행됨: dolly, arc shot, tracking shot, orbital.
  • @image 레퍼런스 태그로 스타트프레임 없이 컷 간 얼굴 일관성.
  • 이전 클립 업로드 → 다음 비트 서술로 씬 연속(얼굴·옷·환경 유지) — 우리 concat 파이프의 연속성 문제 대안.
  • 한 번 생성으로 멀티샷 편집 시퀀스 출력 가능.
  • 3.0 네거티브 프롬프트 미지원은 △(커뮤니티 repo 주장, 공식 블로그엔 언급 없음) — 어차피 포지티브 대안이 안전: "no shaky camera" → "locked-off static camera".
  • 프리셋 라이브러리 = 바이럴 포맷 템플릿 (K-pop 엔딩요정·팬사인회·레드카펫·파파라치·CCTV·나이트비전 등 — 우리 라이프스타일/목격영상 컨셉과 정합).

3. 스타트프레임 엔지니어링 (I2V 품질의 최대 결정 요인)

  1. "스타트프레임 품질 = 포토리얼 결과의 단일 최대 결정 요인. 약한 스타트프레임 = 인공적인 캐릭터" (△ Catacombs 감독, ◐ Kling 공식 동지).
  2. 스틸 프롬프트에 9요소 체계(△ Invideo, 3편 시리즈 전 프레임 유지): 카메라 스펙 → 렌즈&비율 → 조명원(구체 명명) → 팔레트 → 구도 → 대기 → 무드 → 필름/DP 어트리뷰션 → 네거티브.
  3. 스틸을 업스케일한 뒤 I2V에 투입 (해상도 높은 입력 = 모델이 보존할 디테일 증가). 이미지 단계+영상 단계 이중 업스케일이 복리 효과, 이미지 업스케일 생략이 최다 실수 (△).
  4. "cinematic" 같은 추상어 대신 구체 매체 지정: "shot on Sony FX3", "ARRI Alexa 35", "35mm film grain" (△ Genra + 힉스 프리셋 어휘).
  5. 우리 적용: soul_2 스틸(0.12cr) → upscale_image 또는 무료 nano 재생성으로 디테일 강화 → I2V. 실사화 파이프의 nano 스틸도 조명/렌즈 슬롯 의무화.

4. 생성 규율 — 테이크와 체리픽

  • 샷당 예산: ~3 생성 = 1 사용 (keep ~25%). 첫 생성이 그대로 히어로가 되는 일은 없다고 가정 (△ 일치 다수).
  • 프랑켄슈타인 샷(△): 같은 프롬프트 여러 테이크에서 가장 좋은 구간들만 이어붙임. 모션 피크나 프레이밍 일치점에서 컷 → 이음새 안 보임. 같은 프롬프트 골격이라 팔레트/조명 자동 일치.
  • 실패 테이크 채굴: "실패" 클립에도 1~3초 클린 구간 존재 → in/out 마크해 뱅크 (우리 트림 살리기와 동일 — 유지).
  • 구조적 실패(프롬프트 고쳐야) vs 확률적 실패(재롤) 구분해 대응 (reference-prompt-skill DISCIPLINE).
  • 반복은 fast/저가 모드에서, 확정 프롬프트만 고품질 다중 테이크 (비용 통제).
  • 모델 역할 분담(△ Invideo): Kling=네이티브 멀티샷, Veo=시네마틱 단일샷 모션, Seedance 2.0=연속성 엔진(캐릭터+장소 레퍼런스 동시 입력).
  • 우리 예산 적용: 전 샷 3테이크는 과함. 히어로 컷(훅·클라이맥스) 2~3테이크, 나머지 1테이크+게이트 하이브리드. kling3_0_turbo 5s 7.5cr 기준 히어로 3테이크 = 22.5cr/샷.

5. 후반작업 — "AI 광택" 벗기기 (전부 무료 도구로 가능)

순서 고정 (◐/△ 일치): 1. 업스케일 먼저 (그레이드 전에 — 업스케일 후 그레이드가 달리 앉음). 2. 히어로 클립 기준 샷 매칭: AI 클립은 생성마다 화이트밸런스/컨트라스트 제각각+메타데이터 없음 → 웨이브폼/벡터스코프로 뉴트럴·스킨톤 수동 매칭. 3. 프라이머리 그레이드 → LUT는 마지막 노드 (클립 매칭용으로 LUT 쓰지 말 것). 4. 오버샤프 AI 엣지 → 약한 블러로 제거. 5. 필름 그레인 최상층 — 그레인이 최대 리얼리즘 레버(AI 프레임엔 센서 노이즈·유기적 결함이 없고, 그 부재가 곧 '가짜' 인상). 이질적 클립들을 통일하는 매칭 도구이기도. - 프레임 보간(24→60)은 고스팅 리스크 — 기본 업그레이드 아님. 쇼츠는 불필요. - 도구: DaVinci Resolve 무료판으로 전 과정 가능. 우리는 ffmpeg로: eq/curves/colorchannelmixer(그레이드) + haldclut(LUT) + gblur 미세 + noise=alls=6:allf=t+u(그레인) — assemble 스크립트에 표준 패스로 추가. - 업스케일 옵션: 힉스 upscale_video(Topaz, 62s≈41cr — 히어로 편만) / Topaz Astra $299년(AI 푸티지 특화) / CapCut 무료 인핸서(쇼츠엔 충분 △) / SeedVR2 오픈소스(GPU 필요). - 스피드 램핑 10~15%가 둥둥 뜨는 AI 모션 교정 (△). 손 클로즈업은 슬로모 생성 후 배속.

9. 다음 작품 표준 파이프라인 (실행 체크리스트)

  1. 소재 선정 게이트(§16): 트렌드 창(개봉/방영 ±8주) 확인 → 아크 유형(유대·성장·변신=자기완결 우선, 죽음 단독 금지 — 죽음은 3단 압축의 최저점+kama muta 반전 엔딩으로만) → 60초 비트 구조(훅 5%/설정 15%/압축 전개 50%/최저점 20%/감정 정점 13%+루프) 스케치 후 컷시트 착수.
  2. 컷시트 (기존 승인 게이트 유지) + 컷별 프롬프트에 슬롯 의무화: 조명원 / 렌즈·샷사이즈 / 카메라 1무빙+속도+시작끝 / 필름톤 / 네버리스트. +연출 컬럼 4개(§13): 컷 길이(로그노멀 분산, 같은 길이 3연속 금지) / 이 컷이 바꾸는 것 한 단어 / 화면측·시선(L-R 락) / 프레임 오염(clean 3연속 금지). 씬당 투샷 마스터 1컷 배치. +클라이맥스 실측 게이트(§13-F6): 시그니처 액션·클라이맥스는 원작/레퍼런스 프레임을 2~4fps 재추출해 확인한 뒤 설계 — 기억으로 설계 금지.
  3. 스틸: 캐스팅 승인 직후 턴어라운드 시트(3/4·프로필 별도 생성) → 씬은 나노 2k 직출(2cr, 업스케일 단계 폐지) → 투샷 마스터에서 크롭 커버리지 파생 → I2V 투입. 캐치라이트·물기 등 눈 디테일은 스틸 단계에서.
  4. 생성: 기본 엔진 kling3_0 std(7.5cr/5s, sound off), 히어로는 pro. duration은 컷시트 길이대로(3~15s 자유, 홀드 컷 10s). 감정 피크 컷은 first/last frame(표정 B=승인 스틸의 나노 편집본). 모션 프롬프트 3층 표준: 생명 레이어(breathing/blinking)+아크(then+물리행동)+카운터(understated, 네거티브 5~8) — 단 감정 홀드 컷 전용, 액션 컷은 §13-F(모션 동사 문두·양손 서술·의상 명시·holds still 금지). 히어로 컷만 2~3테이크(체리픽·프랑켄슈타인), 일반 컷 1테이크+게이트. Seedance 쓸 땐 fast로 프롬프트 확정 후 std.
  5. 검수: 기존 게이트 A/B 유지 + 모핑 다발 지점(손·머리카락·옷끝·반사면) 확대 검수.
  6. 후반: concat 후 그레이드+미세블러+그레인 ffmpeg 패스(신규 표준) → 히어로 편만 upscale_video 승인 요청.

10. 판타소너 역프롬프트 (2026-07-21, 조회수 상위 3편 프레임 실측)

분석 대상: KDH 촬영유출(19M, 35s), 진격거 실사유출(16M, 39s), 귀멸 무한성 Part 1(7.2M, 34.5s). 전부 2fps 프레임+컷 검출로 실측. 셋 다 "가상 실사화 촬영장 BTS" 포맷.

실측 수치

  • 컷: 8~12개 / 평균 3.3~4.4초. 컷마다 '사건'(마이크로 상호작용)이 있어 4초대도 안 늘어짐.

11. 연출 문법 — 컷이 붙는 이유 (표준 영화 문법 + 우리 실측, 2026-07-21)

프롬프트·후반이 "샷을 잘 만드는 법"이라면 이 장은 "샷을 왜 이 순서로 붙이는가". 표준 영화 문법(고전 이론) + 판타소너/린나이 실측 근거.

  1. 샷 진행 = 정보 공개 순서: Wide(어디)→Medium(누가)→CU(감정)→Insert(디테일)가 기본. 역순(CU 먼저)은 미스터리 훅 — 판타소너 V1 오프닝(의상 조정 CU부터)이 정확히 이것. 컷시트 짤 때 각 컷이 "새로 공개하는 정보"를 한 줄로 적을 것 — 새 정보 없으면 컷하지 말고 같은 샷에서 모션으로.
  2. 컷 온 액션: 동작 중간(모션 피크)에서 자르면 이음새가 안 보임 — 프랑켄슈타인 스티칭이 성립하는 이유. 정지 상태에서 자르면 점프로 보임.
  3. 시선 매칭(eyeline): 인물이 프레임 밖을 보면 다음 컷은 '보는 대상'이어야 하고 시선 방향이 일치해야 함. 180도 규칙: 두 인물을 잇는 축을 카메라가 넘지 말 것 — AI가 리버스샷에서 상습 위반(Catacombs 감독 실측)하므로 프롬프트가 아니라 컷시트에서 방향을 고정(젠이츠 사이드뷰 교훈과 동일 원리).
  4. 모티베이티드 카메라: 무빙엔 이유가 있어야 함 — 따라가기(트래킹)/드러내기(크레인·팬)/다가가기(푸시인=감정 강조). 이유 없는 무빙은 어지러움. Kling 공식 가이드의 "motivated camera movement" 조항과 일치.
  5. J컷/L컷: 다음 컷 소리를 먼저 들려주고(J) 화면을 나중에 바꾸거나, 화면 먼저 소리 나중(L). 컷 이음새를 귀가 덮는다 — 인비저블 컷의 오디오판. ffmpeg에선 오디오 트랙을 컷보다 0.3~0.5s 당겨 배치.
  6. 인서트의 3기능: 정보 전달 + 호흡 조절 + 연속성 붕대(연속성 깨진 두 컷 사이에 매크로 인서트를 끼우면 위화감 소거). 린나이가 컷의 절반을 매크로로 채운 편집적 이유. 우리 파이프에선 아무 데서나 재사용 가능한 "인서트 뱅크"(스팀·물방울·빛·손 클로즈업)를 만들어두면 조립이 빨라짐.
  7. 감정 아크 = 팔레트 시프트: 장(章)이 바뀌면 색이 바뀜 — 린나이 Refresh(쿨 모닝)→Recipe(웜 그린)→Relax(앰버 나이트). 컷시트에 장별 팔레트를 명시하고 그레이드에서 강제.
  8. 스펙터클 배치: 판타소너 실측 — 중반에 1차 리빌, 마지막에 최대 스펙터클. 처음부터 다 보여주면 뒤가 죽는다.

A. 편집 리듬 (딥리서치 2026-07-21 — Murch·Schoonmaker·Cutting ASL 연구·AI 무드릴 비평)

우리 약점 3개(균일 컷·오디오 동시 컷·클라이맥스 무차별)는 문헌이 "AI 무드릴"의 정의로 지목한 바로 그 증상. AI 쇼츠 첫 1분 이탈 40~60% vs 전통 20~30%(Bonthous 실측).

  1. 컷 길이는 로그노멀 분포: 짧은 컷 다수 + 소수의 긴 홀드(현대 영화 ASL 2.5s인데 분포는 비균일 — Cutting/Cornell). 같은 길이 3연속 금지, fast-slow-fast 변주. 컷시트에 길이 열 추가.
  2. 수치 가이드: 악센트 2~3s(빌드 후반 1.5~2s) / 표준 3~5s / 설정 4~7s / 감정 홀드 5~8s+=다른 컷의 2~3배 / 첫 컷 2~4s 안에 첫 전환. AI 얼굴은 ~4s부터 결함 노출 — 결함 클립은 2~3s로 치고 도망, 최고 테이크만 홀드.
  3. 42s 표준 곡선: 훅 3s → 셋업 4컷(3+4+2+4, J컷 연결) → 턴 2s(~18s 지점) → 빌드 5컷 가속(3→1.5s) → 정지 비트 2s(음악 빠지고 앰비언스만) → 클라이맥스 홀드 7s(전체 최장, ~80% 지점) → 릴리즈 4s(첫 컷과 루프). 총 14컷 ASL 3s — 총량은 지금과 같고 체감이 다르다.
  4. Murch Rule of Six: 감정 51 > 스토리 23 > 리듬 10 > 시선 7 > 2D 5 > 3D 4. 버릴 땐 아래부터 — AI 특유의 의상/배경/공간 불일치는 감정이 좋으면 채택(연속성 고치려고 감정 좋은 테이크 버리는 게 최악의 트레이드).
  5. 블링크=컷 포인트: 인물이 깜빡이거나 시선 옮기는 프레임에서 잘라라(AI 클립도 깜빡임 생성됨). 컷 포인트는 산수 말고 실시간 재생으로 — "rightness" 최고점에서 정지(Murch).
  6. 홀드의 기술(Schoonmaker/KOTFM): 감정이 변하는 최고 클립은 통째로 재생, 컷 어웨이는 서사적 동기 있을 때만. 리액션 샷은 아껴서 클라이맥스 직후 1발이 가장 세다. 홀드 컷은 애초에 10s로 생성(kling3_0 duration 자유로 해결).
  7. J/L컷 기본값화: 그림 컷과 소리 컷을 같은 프레임에 두지 마라. 대사/성우 선행 0.5~2s(시작점 1s, 귀로 조정), 교차부 afade 3~5프레임, 음악/앰비언스 브리지는 2s+. 씬 점프 컷엔 무조건 프리랩. 검증=눈 감고 전환부만 듣기(편집이 귀에 들리면 실패). ffmpeg 순서 뒤집기: "오디오 타임라인 먼저, 그림을 거기에 맞춰 컷"(radio edit).
  8. 비트=변화(McKee): 컷시트 각 행에 "이 컷이 바꾸는 것 한 단어" — 못 적으면 예쁜 벽지(렌고쿠 컷1·2 중복이 정확히 이 위반).
  9. 모티프 회귀: 새 쇼트 1개 더 뽑을 크레딧으로 기존 오브젝트(손·눈·검)를 재등장시키는 인서트를 — "아무것도 의미를 가질 만큼 오래 돌아오지 않는" 무드릴 병의 처방.
  10. 끝은 살짝 일찍: "다 됐다 싶은 것보다 조금 일찍 끊어라". 페이오프는 마지막 3s, 엔딩→첫 컷 루프.

B. 공간 연출·스테이징 (딥리서치 2026-07-21 — Hailuo 감독가이드·fal Kling가이드·ShoulderShot 논문 외)

"두 인물 한 프레임"은 2026년 기준 실험 기능이 아니라 주류 워크플로 — 실패는 기능 부재가 아니라 "어느 ref가 누구고 누가 어디 서는지"를 프롬프트에 안 박아서 생긴다.

  1. 정공법 4단계: ①인물별 멀티앵글 ref 시트 → ②나노류 멀티ref로 투샷 마스터 생성(ref-역할 명시: "Image 1 is A: use his face exactly. Do not mix features" + "A on the left, B on the right") → ③마스터에서 크롭/재촬영으로 싱글·OTS 파생 → ④I2V는 미세모션만.
  2. 마스터 1컷이면 족하다: 공간 지리는 한 번 확실히 성립시키면 이후 전부 싱글이어도 "한 공간"으로 읽힘. 편당 투샷 1~2컷이 목표(매 컷 투샷 불필요 — 투샷 0개가 문제였던 것). 클로즈업부터 만드는 게 최대 실수 — 마스터 먼저.
  3. 크롭 커버리지(Crop Duster): 와이드 마스터 1장 → 미디엄/CU/ECU로 점진 크롭 → 각각 업스케일 → 개별 I2V. 한 이미지 파생이라 공간·조명·좌우가 공짜로 일치, 180도 위반 원천 불가. 밋밋한 크롭 OTS는 마스터를 base로 "re-shoot from behind A's shoulder" 재촬영 요청으로 보강.
  4. OTS의 열쇠 — 전경 인물엔 identity ref 넣지 마라(ShoulderShot 논문): 뒤통수는 머리색+헤어 실루엣+의상 텍스트 지정이면 충분, ref를 주입하면 모델이 얼굴 보여주려 구도를 튼다. 패턴: "Over-the-shoulder shot from behind [A], [A]'s blurred shoulder and back of head in the foreground left, camera focused on [B]'s face, shallow depth of field, 85mm". OTS는 구도 이해도는 높은데 미학 일관성 최하위 — 테이크 2배 배정.
  5. 9:16 세로 투샷은 좌우가 아니라 앞뒤 스택: "[A] in foreground, face fills upper two-thirds; [B] at 1.5-meter depth behind, visible over her left shoulder" — 깊이 명시로 세로 프레임에 두 인물 공존.
  6. 180도 룰=시선의 수학: A의 전 샷 "looking slightly camera-right", B는 "camera-left" — 이 두 줄로 컷을 붙이면 마주보는 걸로 읽힘. "looking at the other person" 같은 해석형 지시 금지. 시선 반대로 나오면 재생성 말고 좌우 플립(단 가르마·문양·글자 비대칭 주의 — 캐스팅 때 비대칭 최소화하면 플립 자유도 확보).
  7. 조명 절대값 반복: "key light from camera-left, warm amber"를 매 샷 프롬프트에 복사 — 안 하면 리버스샷에서 그림자가 뒤집혀 공간이 깨짐.
  8. 두 인물 I2V 사고 방지: 엉뚱한 인물이 움직이는 게 표준 실패 — 인물 명명+행동 배분 명시("The black-suited agent slams the table; the woman stays still"). 대사 있으면 행동 먼저→발화 귀속.
  9. start+end 프레임 응용: 시작=투샷 마스터, 끝=싱글 → 컷 없이 두 구도를 잇는 카메라 이동 생성 = "같은 공간 증명"의 최저가 수단. 같은 마스터 파생 프레임끼리만(색/조명 다르면 워프 깨짐). 편당 1회면 공간 인상이 전체에 깔림.
  10. 더러운 프레임: clean frame 3연속 금지 — 상대의 흐린 어깨, 문틀, 풀숲을 프레임에 걸쳐라. 세로는 하단 1/4을 전경 오브젝트로. POV는 "[A]'s hand visible at bottom of frame reaching toward..."로 즉시 성립.
  11. 3층 피라미드: 전경(질감·아웃포커스)/중경(피사체·액션)/배경(축약 토큰만) — 셋 다 상세히 쓰면 전 층에 초점 걸려 평평해짐. 파랄랙스는 "slow parallax drift" 명시.
  12. 비정면 각도 붕괴 순서: 정면(안정)→3/4(대체로 유지)→프로필(코·턱 드리프트)→후면(창작됨). 해법=캐스팅 승인 직후 턴어라운드 시트(3/4 좌우·프로필을 각각 별도 생성, 한 장에 몰지 말 것) → 이후 샷마다 해당 각도 ref 사용. 프로필 샷에 정면 ref만 주는 게 실패의 근원. 로우앵글 히어로/하이앵글 취약 앵글 자체는 표준 어휘로 잘 나옴 — 문제는 ref 커버리지.
  13. 렌즈로 참여감 조절(코엔/디킨스): 같은 솔로 CU도 "wide-angle lens, close to subject"=대화 사이에 앉은 참여감 vs "85mm compressed"=관찰자. 망원 초상화 룩만 반복되는 게 죄.
  14. 컷시트 컬럼 3개 추가: 화면측(L/R) | 시선(camera-left/right+각도) | 프레임 오염(clean/dirty: 뭐가 걸치는가). 씬 헤더에 축 선언("A=좌, B=우 불변")+키라이트 방향+마스터 파일명.

C. AI 연기 디렉션 (딥리서치 2026-07-21 — fal/Kling 공식·Curious Refuge·Segmind·videoai.me 실측)

핵심: 모델은 "상태"가 아니라 "변화"를 시켜야 움직인다. 표정이 얼어붙는 건 감정 라벨(sad)만 있고 물리 변화 동사+순서어가 없어서. Kling 3.0은 시키면 오버액팅, 안 시키면 정지 — 절제된 마이크로 연기는 명시 주문해야만 나온다(Curious Refuge 실측).

  1. 기본 생명 레이어(전 클립 상수): 모션 프롬프트 끝에 "Breathing motion, blinking eyes" 또는 "Subtle breathing, gentle blinking, slight head movement, hair shifting softly" — fal 공식 예문의 상비 토큰. 렌고쿠 연기 아크 부재의 직접 처방.
  2. 감정=물리 번역: ❌"sad" → ⭕"tears rolling down cheek", "raising his eyebrows slightly", "breathing quickens". 감정+물리행동 페어가 감정 단독보다 우수(Segmind 실측). 놀람/충격 계열이 과장 1순위 — understated 토큰 필수.
  3. 오버액팅 카운터(상비): "natural, understated performance, subtle facial expressions, no exaggerated acting" + 네거티브 5~8개(중요순 앞배치, 20개+는 연기가 죽음): frozen lips, jittery eyes, unnatural blinking, plastic skin, head shake, exaggerated expression. 동작 종결어 "then settles back into place"로 폭주 방지.
  4. 아크 패턴: [홀드 상태] → then/suddenly → [도착 표정+물리 디테일]. 예: "She holds still, then her lip trembles and a single tear rolls down her cheek." 순서어 First/then/finally/Pause를 Kling이 존중. 5s=전환 1개(홀드 2s→턴 3s), 10s=홀드→턴→정착 3박까지만(후반 드리프트 리스크). 감정 피크 컷만 10s, 나머지 5s.
  5. 스타트프레임 표정 룰: 뉴트럴 인접에서 시작해 헤드룸 확보(Kling 공식 페어가 전부 낮은 감정→높은 감정 방향: Calm→surprised, Neutral→emotional). 예외=눈물 컷: 물기 이미 맺힌 스틸("half-closed watery eyes, damp clumped eyelashes")이어야 5초 안에 눈물이 착지. 스틸에 있는 건 재묘사 금지(변화만 서술 — 기존 원칙 유지).
  6. first/last frame = 표정 전환 강제 공식 도구: 표정 B 프레임은 새로 뽑지 말고 승인 스틸 A를 나노 이미지 편집으로 표정만 바꿔 제작(동일 인물·조명·앵글 자동 보장). 두 프레임은 같은 앵글·조명·색감 필수(다르면 렌즈 스위치/플리커), 프롬프트는 라이트 터치: "subtle facial morph; keep eye line steady; minimal camera sway". 전환은 5s가 클린.
  7. 눈: 캐치라이트는 I2V가 아니라 스틸 단계에서 박는다("catchlights, detailed iris texture, sharp eye focus"). 물기는 텍스트로 됨("eyes glistening with emotion"). 시선 방향 텍스트 지시는 신뢰도 낮음 → 카메라가 시선을 따라가게 하거나 모션컨트롤로.
  8. 퍼포먼스 트랜스퍼(감정 CU 최강 레버): Kling Motion Control이 입가·시선·미간까지 전이 — 본인 셀프캠 3~10s 연기로 텍스트 불가능한 연기 주입. 조건: 원컷·무빙 금지·1인·프레이밍 스케일 매칭(CU 연기엔 CU 셀프캠 — 풀바디 ref를 초상에 쓰면 얼굴 워프), 몸 동작 크면 얼굴 드리프트. 드라이버=실사 필수(기존 실측 일치, 애니 드라이버 ❌).

D. 경쟁 채널 실측 — 판타소너 스토리 쇼츠 문법 (2026-07-21, yt-dlp 프레임 분석. 로컬 사본 rengoku/refs/competitors/)

결정적 사실: 판타소너도 렌고쿠 스토리를 만들었다(shorts/pWt7jX-3LJA, 2026-02-17, 60s) — 12.7만 회. 같은 채널 미츠리(236만)·레제(340만)의 1/20. 소재·엔딩 컨셉(새벽 들판+어머니 환영)이 우리와 동일해 직접 비교 교본.

  1. 투샷이 커버리지의 절반+: 모자 정좌 프로필 투샷 → 포옹 CU 투샷 → 부자 투샷(랜턴 로우키) → 미츠리 수련 투샷 → 사탕가게 유대 투샷 → 아카자 대치 와이드. 공간 관계가 항상 프레임 안에서 성립 — 우리 진단 1번(투샷 0개)의 정답지.
  2. 정면 초상 반복 없음: 프로필·백샷(뒷모습)·오버헤드·문틀/장지문 프레이밍 로테이션. 정면은 ①부상 홀드(7s+, 최장 컷) ②엔딩 미소 딱 2회 — 아껴서 씀. 단 OTS 리버스 편집은 이 채널도 회피(아이라인 리스크).
