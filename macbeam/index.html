왜 MacBeam?

⚡

60fps 저지연 미러링

VideoToolbox H.264 하드웨어 인코딩. 네트워크 약하면 자동으로 fps/bitrate 조절.

⌨️

한글·영문·이모지 전부

한글 IME 직접 동기화. ⌘ ⌥ ⌃ ⇧ 단축키 스티키 모드. 자주 쓰는 단축키 10개 popup 저장.

🔊

시스템 오디오

Mac 의 모든 소리가 폰으로. 세션 중에는 Mac 스피커 자동 음소거, 종료 시 자동 복원.

🔒

P2P + TLS 1.2+

외부 중계 서버 없음. TLS 인증서 핀고정 (TOFU) 으로 MITM 차단. 32B PSK + HMAC.

👆

트랙패드 제스처

1손가락 = 좌클릭, 2손가락 탭 = 우클릭, 2손가락 스크롤, 핀치 줌. 더블탭 + 홀드 = 드래그.

🌐

VPN 호환

Tailscale, WireGuard, ZeroTier — 어디서든 두 기기가 같은 가상망에 있으면 작동.

접속 완전 가이드

처음 쓰시는 분도 따라하면 됩니다. 상황에 맞는 방법을 고르세요.

📶 방법 1 — 같은 Wi-Fi에서 연결하기

가장 기본. iPhone 과 Mac 이 같은 공유기에 연결돼 있으면 됩니다.

1

Mac 서버 앱 설치

아래 버튼으로 DMG 파일을 다운로드하고, 열면 나오는 창에서 MacRemoteServer.app을 오른쪽 Applications 폴더로 드래그하세요.

⬇ MacBeam-Server-v0.3.0.dmg 다운로드

💡 macOS 가 "확인되지 않은 개발자" 경고를 보이면: 시스템 설정 → 개인정보 보호 및 보안 → 하단 '확인 없이 열기' 클릭 (Apple notarize 된 앱이라 안전합니다)
2

MacBeam Server 실행 + 권한 허용

Applications 폴더에서 MacRemoteServer를 더블클릭해서 실행하세요. 메뉴바(오른쪽 위 시계 옆) 에 🔒🛡️ 아이콘이 나타납니다.

처음 실행 시 권한 요청이 뜹니다. 두 가지 모두 허용해야 합니다:

• 손쉬운 사용 — 마우스·키보드 제어용
• 화면 기록 — 화면 미러링용

💡 권한 창이 안 뜨면: 시스템 설정 → 개인정보 보호 및 보안 → 손쉬운 사용에서 MacRemoteServer 를 직접 추가하세요
3

iPhone에 MacBeam 앱 설치

App Store 에서 MacBeam을 검색해서 설치하세요. (또는 출시 알림 이메일의 링크 이용)

4

PIN 페어링으로 연결

Mac 에서: 메뉴바의 🔒🛡️ 아이콘 클릭 → "PIN 페어링 시작 (60초)" 클릭 → 6자리 숫자가 팝업으로 표시됩니다.

iPhone 에서: MacBeam 앱 → "Wi-Fi 로 자동 찾기" → 목록에서 Mac 이름 선택 → 6자리 PIN 입력 → 페어링 완료.

💡 Mac 이름이 목록에 안 보이면? iPhone 과 Mac 이 정말 같은 Wi-Fi 인지 확인. 다른 네트워크(LTE vs Wi-Fi 등)면 안 보입니다.
5

완료! 다음부터는 자동 연결

페어링은 딱 1번만 합니다. 이후에는 같은 Wi-Fi 에서 앱을 열고 "연결"만 누르면 됩니다. Mac 서버는 재부팅해도 자동으로 실행됩니다.

🌍 방법 2 — 집 밖에서 연결하기 (Tailscale)

카페·직장·지하철 어디서든. 무료 앱 Tailscale 하나면 됩니다. 5분 셋업.

1

방법 1 (같은 Wi-Fi)의 1~4단계를 먼저 완료하세요

처음 페어링은 반드시 같은 Wi-Fi 에서 진행해야 합니다. 이미 페어링을 마쳤다면 바로 2단계로.

2

Mac 에 Tailscale 설치

tailscale.com/download → macOS 버전 다운로드 → 설치 후 실행 → Google·Apple·Microsoft 계정 중 하나로 로그인.

로그인하면 메뉴바에 Tailscale 아이콘이 생기고, 100.x.x.x 형식의 가상 IP 가 부여됩니다.

💡 이 100.x.x.x IP 를 기억해 두세요. MacBeam 에서 이 주소로 접속합니다.
3

iPhone 에 Tailscale 설치

App Store 에서 Tailscale 검색 → 설치 → Mac 과 같은 계정으로 로그인 → VPN 허용 → 연결 켜기.

Tailscale 앱에서 Mac 이름이 목록에 보이면 준비 완료입니다.

4

MacBeam 에 Tailscale IP 입력

MacBeam 앱 → 설정(⚙️) → Host 에 Mac 의 100.x.x.x 주소 입력 → Port 는 5901 유지 → 완료.

이제 집 밖 어디서든 MacBeam 앱 → "연결" 만 누르면 됩니다.

💡 두 기기 모두 Tailscale VPN 이 켜진 상태여야 합니다. Mac 에서 Tailscale 이 꺼져 있으면 연결 안 됩니다.
⚠️ 통신사 CGNAT 환경(일부 LTE/5G)에서도 Tailscale 은 정상 동작합니다. 공인 IP 필요 없음.

⏻ 방법 3 — 잠든 Mac 깨우기 (Wake-on-LAN)

Mac 이 절전 상태일 때 iPhone 에서 원격으로 켤 수 있습니다.

1

Mac 에서 '네트워크 접근으로 깨우기' 켜기 필수

시스템 설정 → 배터리 (또는 에너지 절약) → "네트워크 접근으로 깨우기" 토글 ON

Wi-Fi 로 연결된 Mac 이라면 "Wi-Fi 네트워크 접근으로 깨우기" 항목도 ON 하세요.

⚠️ Mac 을 멀티탭에서 완전히 차단했거나 배터리가 방전된 MacBook 은 깨울 수 없습니다. 절전(수면) 상태여야 합니다.
2

서버 v0.3.0 이상으로 재페어링 WOL v0.3.0+

WOL 기능은 Mac 서버 v0.3.0 부터 지원합니다. 기존에 더 낮은 버전으로 페어링했다면, 서버를 업데이트하고 PIN 페어링을 한 번 더 진행해야 Mac 의 MAC 주소가 iPhone 에 저장됩니다.

서버 업데이트: MacBeam-Server-v0.3.0.dmg 재설치 → 재실행 → PIN 페어링 1회.

3

같은 Wi-Fi에서 깨우기 자동

페어링을 마치면 MacBeam 앱 유휴 화면에 "⏻ Mac 깨우기" 버튼이 나타납니다.

Mac 이 절전 상태일 때 이 버튼을 누르면 매직 패킷이 전송됩니다. 보통 5~30초 후 Mac 이 켜지면 "연결" 버튼으로 접속하면 됩니다.

💡 연결 시도 시 Mac 이 꺼져 있으면 앱이 자동으로 매직 패킷을 1회 보내고 재연결을 시도합니다.
4

집 밖에서 깨우기 — 공유기 포트포워딩 고급

다른 네트워크(LTE, 다른 집 Wi-Fi)에서도 잠든 Mac 을 깨울 수 있습니다. 공유기 설정이 필요합니다.

공유기 설정:
관리자 페이지(보통 192.168.0.1 또는 192.168.1.1) → 포트포워딩/NAT
• 프로토콜: UDP
• 외부 포트: 9
• 내부 IP: Mac 의 내부 IP (시스템 설정 → Wi-Fi → ⓘ 에서 확인)
• 내부 포트: 9

MacBeam 설정:
설정(⚙️) → "Wake-on-LAN 원격 (포트포워딩)" → 공유기 공인 IP (또는 DDNS) 와 포트 9 입력 → 저장.

💡 집 IP 가 자주 바뀐다면 공유기의 DDNS 기능을 켜서 고정 주소(예: home.iptime.org)를 만드세요.
⚠️ 통신사 CGNAT 환경(공인 IP 없음)에서는 포트포워딩이 동작하지 않습니다. 이 경우 Tailscale 을 사용하세요.

🔧 안 될 때 체크리스트

대부분의 문제는 아래 순서로 해결됩니다.

✓

Mac 서버가 실행 중인가요?

메뉴바(화면 오른쪽 위)에 🔒🛡️ 아이콘이 보이는지 확인하세요. 없으면 Applications 에서 MacRemoteServer 를 다시 실행하세요.

✓

권한이 모두 허용됐나요?

시스템 설정 → 개인정보 보호 및 보안
→ 손쉬운 사용: MacRemote Server 체크 ✓
→ 화면 기록: MacRemote Server 체크 ✓
둘 중 하나라도 꺼져 있으면 연결은 되지만 화면이 안 나옵니다.

✓

같은 Wi-Fi인데 Mac 이 목록에 안 보여요

• iPhone 이 LTE/5G 로 연결된 건 아닌지 확인 (Wi-Fi 아이콘 확인)
• Mac 에서 MacRemote Server 를 종료했다가 다시 실행
• iPhone Wi-Fi 를 껐다가 다시 켜기
• 회사·학교 네트워크는 기기 간 통신을 차단하는 경우가 많음 → Tailscale 사용

✓

연결은 되는데 화면이 검은색으로 나와요

화면 기록 권한이 없거나, macOS 잠금 화면 상태일 때 발생합니다.
→ 시스템 설정 → 화면 기록 에서 MacRemote Server 권한 확인
→ Mac 잠금 화면을 먼저 해제한 뒤 연결 시도

✓

TLS 인증서 불일치 오류 (Mac 교체·재설치 후)

Mac 을 교체하거나 macOS 를 클린 설치했을 때 발생합니다.
MacBeam 앱 → 설정 → "다른 Mac 으로 페어링" 을 눌러 처음부터 다시 페어링하세요.

✓

그래도 안 되면

위 방법을 모두 시도해도 해결 안 되면 언제든 문의해 주세요. 한국어 응대 가능합니다.

자주 묻는 질문

Mac 서버는 왜 따로 설치해야 하나요?

점프데스크탑 / Screens / Splashtop 처럼 화면을 보낼 Mac 쪽에 작은 헬퍼 앱이 필요합니다. 무료 + 오픈 소스이고, 한 번만 설치하면 메뉴바에 자물쇠+방패 아이콘만 남아 보이지 않게 동작합니다.

외부 서버를 거치나요?

아니요. iPhone ↔ Mac 사이에 TLS 1.2+ P2P 연결만 합니다. 외부 중계 서버는 없으며, 데이터는 사용자 네트워크 밖으로 나가지 않습니다.

집 밖에서도 쓸 수 있나요?

네 — Tailscale, WireGuard, ZeroTier 같은 VPN 으로 두 기기를 연결하면 어디서든 작동합니다. Tailscale 은 무료 + 5분 셋업.

VNC 와 뭐가 다른가요?

macOS 기본 VNC 는 압축 없는 비트맵 — 같은 LAN 에서도 느림. MacBeam 은 H.264 하드웨어 인코딩으로 60fps 저지연 + 시스템 오디오 + 한글 IME 동기 + 우리만의 단축키/제스처 popup.

구독 취소는?

iPhone 설정 → Apple ID → 구독 → MacBeam → 취소. 다음 결제일 24시간 전까지. 현재 주기 종료까지는 사용 가능.
