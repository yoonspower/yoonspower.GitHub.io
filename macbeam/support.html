문제 해결과 문의를 한 곳에서.

자주 묻는 질문

Mac 서버 앱은 어디서 받나요?

MacBeam 은 iPhone 앱과 Mac 서버 앱이 한 쌍으로 동작합니다. Mac 서버는 무료로 배포됩니다.

다운로드: MacBeam-Server-v0.4.0.dmg (189 KB, notarized)

다운받은 DMG 를 열고 MacBeam Server (실행 파일명: MacRemoteServer.app — 프로토콜 호환성을 위해 v1 이름 유지) 를 Applications 폴더로 드래그하면 설치 완료. macOS 14 (Sonoma) 이상 필요.

페어링이 안 되는데요?

1. iPhone 과 Mac 이 같은 Wi-Fi 또는 같은 VPN (Tailscale 등) 에 있는지 확인하세요.
2. Mac 메뉴바의 🔒🛡️ 아이콘이 켜져 있는지 확인하세요.
3. 시스템 설정 → 손쉬운 사용 + 화면 기록 권한이 MacBeam Server (MacRemoteServer) 에 체크되어 있는지 확인하세요.
4. 방화벽이 켜져 있으면 MacBeam Server (MacRemoteServer) 를 허용해주세요.

화면이 끊기거나 느려요

앱 → 설정 → "해상도 / FPS" 에서 낮은 값으로 조정해보세요. 네트워크가 약하면 자동으로 fps/bitrate 가 조절되지만, 수동으로 720p / 30fps 등을 고정하면 더 안정적입니다.

한글 입력이 안 돼요

iPhone 의 키보드를 한글로 전환한 상태에서 입력하면 자동으로 Mac 의 한글 IME 와 동기화됩니다. 그래도 안 되면 페어링 초기화 후 재페어링을 시도해보세요.

구독 취소는 어떻게 하나요?

iPhone 설정 → Apple ID → 구독 → MacBeam → 취소. 다음 결제일 24시간 전까지 취소 가능하며, 현재 결제 주기 종료까지는 사용 가능합니다.

환불은 어떻게 받나요?

모든 결제는 Apple App Store 를 통해 처리됩니다. 환불은 reportaproblem.apple.com 에서 신청할 수 있으며, 승인 여부는 Apple 정책에 따릅니다.

광고가 너무 자주 나와요

첫 설치 후 8시간 무광고로 사용 가능합니다. 이후 광고 1회 시청당 추가 4시간 (최대 24시간) 무광고가 누적됩니다. 영구 광고 제거는 월정액 ₩1,900 또는 평생 ₩10,000 결제 시 가능합니다.

"페어링 초기화" 는 무엇을 지우나요?

저장된 Mac 호스트, 사전 공유 키 (PSK), TLS 인증서 지문이 모두 삭제됩니다. 다음 연결 시 페어링을 처음부터 다시 진행해야 합니다.

집 밖에서 원격 접속하기

MacBeam 은 기본적으로 iPhone 과 Mac 이 같은 Wi-Fi 에 있을 때 자동으로 연결됩니다. 외출 중에도 집 Mac 에 접속하려면, 두 기기를 같은 가상 네트워크로 묶어주면 됩니다. 가장 쉬운 방법은 무료 앱 Tailscale 입니다.

Tailscale 로 어디서나 접속 (초보자 추천 · 무료)

1. Mac 에 Tailscale 설치: tailscale.com/download → 실행 후 Google/Apple/Microsoft 계정으로 로그인.
2. iPhone 에 App Store 에서 Tailscale 설치 → 같은 계정 으로 로그인하고 VPN 을 켭니다.
3. 두 기기 모두 Tailscale 이 켜진 상태가 되면, Tailscale 앱에서 Mac 의 IP (보통 100.x.x.x) 를 확인하세요.
4. 처음 한 번은 같은 Wi-Fi 에서 페어링 을 끝내 두는 것을 권장합니다 (PIN 페어링으로 인증키 저장).
5. 이후 외부에서는 MacBeam 앱 → 설정 → "호스트 IP" 에 위 100.x.x.x 주소, 포트 5901 을 입력하면 바로 연결됩니다.

Tailscale 무료 플랜으로 개인 기기 다수를 연결할 수 있고, 카페·LTE·다른 집 어디서든 (통신사 CGNAT 환경 포함) 동작합니다. MacBeam 은 중계 서버를 두지 않는 순수 P2P 방식이라, 이렇게 두 기기를 같은 네트워크로 묶어 주는 단계가 필요합니다.

고급: VPN 없이 포트 포워딩으로 접속하기

네트워크에 익숙한 사용자라면 공유기 설정으로도 가능합니다 (공인 IP 가 있는 경우).

1. 같은 Wi-Fi 에서 먼저 1회 페어링.
2. 공유기 관리자 → 포트 포워딩: 외부 포트(예: 5901) → Mac 내부 IP 의 5901/TCP.
3. 집 IP 가 바뀌므로 공유기 DDNS 를 켜서 고정 주소를 만듭니다.
4. MacBeam 앱 → 설정 → 호스트 IP 에 공인 IP 또는 DDNS 주소, 포트에 외부 포트를 입력.

⚠️ 화면 제어 포트를 인터넷에 여는 것이므로, 비표준 포트 사용을 권장합니다. 연결은 TLS + 사전 공유 키(PSK) + 인증서 핀고정으로 보호됩니다. 통신사가 CGNAT 를 쓰는 경우 이 방법은 동작하지 않으니 Tailscale 을 사용하세요.

다른 네트워크에서 잠든 Mac 깨우기 (WOL 포트포워딩)

Wake-on-LAN(WOL)은 매직 패킷을 Mac 의 NIC(랜카드)에 보내 전원을 켜는 기능입니다. 같은 Wi-Fi 에서는 앱이 자동으로 처리하지만, 집 밖에서도 잠든 Mac 을 깨우려면 공유기에 포트포워딩 설정이 필요합니다.

🔧 원리

iPhone (외부 네트워크) → 공유기 공인 IP:UDP 9 → 공유기 포트포워딩 → Mac 내부 IP:UDP 9 → Mac NIC 수신 → 전원 ON

Mac 이 꺼져 있어도 NIC 는 매직 패킷을 감지할 수 있습니다. 단, Mac 이 완전 전원차단(멀티탭 OFF 등)이면 동작하지 않습니다 — 절전/수면 상태여야 합니다.

📋 설정 순서

1. Mac — '네트워크 접근으로 깨우기' 활성
  시스템 설정 → 배터리 (또는 에너지 절약) → "네트워크 접근으로 깨우기" 토글 ON
  Wi-Fi 연결이면 "Wi-Fi 네트워크 접근으로 깨우기" 항목도 ON

2. Mac 내부 IP 고정
  시스템 설정 → Wi-Fi → 연결된 네트워크 옆 ⓘ → TCP/IP → DHCP 임대 갱신 후 IP 확인.
  공유기 관리 페이지에서 해당 MAC 주소에 IP 고정(DHCP 고정 할당) 설정 권장.

3. 공유기 포트포워딩 설정
  공유기 관리 페이지(보통 192.168.0.1 또는 192.168.1.1) → 포트포워딩/NAT
  • 프로토콜: UDP
  • 외부 포트: 9
  • 내부 IP: Mac 의 내부 IP (예: 192.168.0.42)
  • 내부 포트: 9
  포트 9 가 막혀있으면 7 또는 임의 포트로 대체 가능.

4. 공인 IP 또는 DDNS 확인
  공유기 관리 페이지 → WAN 정보에서 공인 IP 확인.
  집 IP 가 수시로 바뀐다면 공유기의 DDNS 기능을 켜서 고정 주소(예: home.iptime.org) 발급.

5. MacBeam 앱 설정
  MacBeam → 설정 → "Wake-on-LAN 원격 (포트포워딩)" 섹션
  • 공인 IP / DDNS: 위에서 확인한 주소 입력
  • UDP 포트: 9 (또는 공유기에 설정한 포트)
  → 저장 후 "Mac 깨우기" 버튼을 누르면 외부에서도 전송됩니다.

⚠️ 주의사항

• 통신사 CGNAT 환경(공인 IP 없음, LTE/5G 일부)에서는 포트포워딩이 동작하지 않습니다. 이 경우 Tailscale 을 사용하세요.
• 공유기 모델에 따라 UDP 브로드캐스트 포워딩을 지원하지 않는 경우가 있습니다. 안 될 경우 Tailscale 방법을 시도하세요.
• Mac 전원 케이블이 연결되어 있어야 WOL 이 동작하는 기종이 많습니다 (MacBook 배터리 방전 상태 등).
• 깨어나는 데 보통 5~30초가 걸립니다. "Mac 깨우기" 버튼 누른 후 잠시 기다렸다가 "연결" 을 시도하세요.
